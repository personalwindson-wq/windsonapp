## Directory Purposes

**Root Level:**
- Purpose: Project configuration, entry points, build scripts
- Contains: `index.html`, `manifest.json`, `sw.js`, `config.js`, `package.json`, etc.

**`src/` (Source Code):**
- Purpose: All application source code
- Contains: JavaScript modules, CSS source, utilities

**`src/modules/` (Business Logic):**
- Purpose: Feature modules with business logic and UI manipulation
- Contains: auth.js, avaliacao.js, exerciseBank.js, ui.js

**`src/services/` (External Communication):**
- Purpose: API clients and data operations
- Contains: supabaseService.js, rapidApiService.js

**`src/data/` (Static Reference Data):**
- Purpose: Static lookup tables and protocols
- Contains: avProto.js, exEnMap.js

**`src/utils/` (Helper Functions):**
- Purpose: Pure utility functions
- Contains: geometry.js

**`src/css/` (Styles):**
- Purpose: Tailwind CSS source + custom styles
- Contains: input.css

**`dist/` (Generated):**
- Purpose: Compiled CSS output
- Contains: style.css (generated by Tailwind CLI)

**`scripts/` (Build/Dev Utilities):**
- Purpose: Database setup, data import, GIF processing
- Contains: SQL files, Node.js scripts

## Naming Conventions

**Files:**
- camelCase: `supabaseService.js`, `rapidApiService.js`
- camelCase modules: `exerciseBank.js`, `avaliacao.js`
- kebab-case for CSS: N/A (Tailwind)
- snake_case for SQL: `migration.sql`, `seed_exercises.sql`

**Directories:**
- kebab-case: `src/modules/`, `src/services/`, `src/utils/`

**Functions/Variables:**
- camelCase: `fetchClientes`, `exBankFilter`, `avRunAnalysis`
- PascalCase for constructors: N/A (no classes)
- SCREAMING_SNAKE_CASE for constants: `ALLOWED_EMAILS`, `CACHE_NAME`

**Types:**
- No TypeScript - plain JS
- Object shape documented in comments

**HTML IDs:**
- kebab-case: `tab-dashboard`, `clientes-grid`, `av-upload-panel`
- Semantic prefixes: `av-*` (assessment), `ex-*` (exercise), `sw-*` (save workout)

**CSS Classes:**
- Tailwind utility classes in HTML
- Custom classes: kebab-case in CSS (`glass-panel`, `skeleton-shimmer`)

## Where to Add New Code

**New Feature/Module:**
- Primary code: `src/modules/{featureName}.js`
- Export functions via `window.{functionName} = ...`
- Import service layer: `const sbClient = window.sbClient`

**New Database Table:**
- Add schema to `scripts/migration.sql`
- Add CRUD functions to `src/services/supabaseService.js`
- Expose via `window.{operation}`

**New Assessment Protocol:**
- Add to `src/data/avProto.js` following existing structure
- Key format: `metricKey: { label, mild: [...], moderate: [...], ci: [...] }`

**New Exercise Mapping:**
- Add to `src/data/exEnMap.js` EX_EN_MAP object
- Format: `'Portuguese Name': 'english-term'`

**New UI Tab:**
- Add section to `index.html` with `id="tab-{name}"` and class `tab-section`
- Add navigation button with `data-tab="tab-{name}"`
- Add colors to `_meshColors` in `src/modules/ui.js:22-29`
- Add tab activation in `showTab()` function
